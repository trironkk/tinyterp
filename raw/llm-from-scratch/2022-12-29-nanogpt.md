## What it is

nanoGPT is "the simplest, fastest repository for training/finetuning medium-sized GPTs."
It is a rewrite of [minGPT](https://github.com/karpathy/minGPT) that "prioritizes teeth over
education." The code is deliberately plain and readable: `train.py` is a ~300-line boilerplate
training loop and `model.py` a ~300-line GPT model definition that can optionally load the
GPT-2 weights from OpenAI. Because the code is so simple, it is easy to hack — train new models
from scratch or finetune pretrained checkpoints.

> **Update Nov 2025:** nanoGPT has a new and improved cousin called
> [nanochat](https://github.com/karpathy/nanochat). nanoGPT is now considered old/deprecated but
> kept up for posterity.

## Capabilities

- `train.py` reproduces GPT-2 (124M) on OpenWebText, running on a single 8×A100 40GB node in
  about 4 days of training, reaching a loss of ~2.85.
- Dependencies: PyTorch, numpy, `transformers` (to load GPT-2 checkpoints), `datasets` (to
  download/preprocess OpenWebText), `tiktoken` (OpenAI's fast BPE), `wandb` (optional logging),
  `tqdm`.

## Quick start (character-level Shakespeare)

The fastest on-ramp is a character-level GPT on the works of Shakespeare (a single 1MB file
turned into a stream of integers via `data/shakespeare_char/prepare.py`).

- **With a GPU:** the provided config trains a 6-layer, 6-head Transformer with 384 feature
  channels and context size 256. On one A100 this takes ~3 minutes and reaches best validation
  loss 1.4697. Sample with `sample.py --out_dir=out-shakespeare-char`.
- **On a Macbook/CPU:** dial things down (`--device=cpu --compile=False`, smaller block size,
  batch size, and a 4-layer/4-head/128-embd model, dropout 0). Still ~3 minutes, loss ~1.88.
  Apple Silicon can use `--device=mps` for a 2–3× speedup.

## Reproducing GPT-2

Tokenize OpenWebText (`data/openwebtext/prepare.py`) into `train.bin`/`val.bin` of GPT-2 BPE
token ids stored as raw uint16, then launch distributed training:

```sh
torchrun --standalone --nproc_per_node=8 train.py config/train_gpt2.py
```

This runs ~4 days via PyTorch Distributed Data Parallel (DDP) down to loss ~2.85. Multi-node
training is supported via `torchrun` rendezvous flags (benchmark your interconnect; prepend
`NCCL_IB_DISABLE=1` without Infiniband).

## Baselines (OpenAI GPT-2 checkpoints on OpenWebText)

| model | params | train loss | val loss |
|-------|--------|------------|----------|
| gpt2 | 124M | 3.11 | 3.12 |
| gpt2-medium | 350M | 2.85 | 2.84 |
| gpt2-large | 774M | 2.66 | 2.67 |
| gpt2-xl | 1558M | 2.56 | 2.54 |

GPT-2 was trained on the closed WebText; OpenWebText is a best-effort open reproduction, so a
domain gap exists. Finetuning GPT-2 (124M) on OWT reaches ~2.85, which becomes the more
appropriate reproduction baseline.

## Finetuning, sampling, efficiency

- **Finetuning** is just training initialized from a pretrained checkpoint (`init_from`) with a
  smaller learning rate; e.g. finetune on tiny Shakespeare via `config/finetune_shakespeare.py`.
- **Sampling/inference** via `sample.py` from OpenAI GPT-2 models or your own checkpoints
  (`--init_from=gpt2-xl`, `--start=...`, `--num_samples`, `--max_new_tokens`; prompt from a file
  with `--start=FILE:prompt.txt`).
- **Efficiency:** uses PyTorch 2.0 `torch.compile()` by default — a one-line change that cut
  iteration time from ~250ms to ~135ms. `bench.py` is available for profiling.
