# MicroGPT: A Minimal Language Model Implementation

Andrej Karpathy's blog post introduces microGPT, a ~200-line Python project that distills GPT training and inference into its essential components without external dependencies.

## Core Components

The implementation covers several key sections:

**Dataset & Tokenization**: The model trains on 32,000 names, converting text to token IDs using a character-level tokenizer with 27 tokens (26 lowercase letters plus a special BOS marker).

**Autograd Engine**: A custom `Value` class implements automatic differentiation by tracking computation graphs and applying the chain rule during backpropagation — functionally identical to PyTorch's approach but operating on scalars.

**Architecture**: The model follows a GPT-2-style Transformer with:
- Token and position embeddings
- Multi-head attention for token communication
- MLP blocks for per-position computation
- Residual connections
- RMSNorm normalization

**Training Loop**: The system iteratively (1) tokenizes documents, (2) runs forward passes building computation graphs, (3) computes cross-entropy loss, (4) backpropagates gradients, and (5) updates parameters using the Adam optimizer with learning rate decay.

**Inference**: After training, the frozen model generates new names by sampling tokens autoregressively from learned probability distributions, controlled by a temperature parameter.

## Key Insight

The author emphasizes that "everything else is just efficiency" — the 200 lines contain the complete algorithmic essence. Production systems scale this foundation through larger datasets, subword tokenizers, tensor operations on GPUs, and extensive post-training procedures, but the core mathematics remains unchanged.

The project includes progressive training files (e.g., train0.py through train5.py) showing incremental feature additions.
