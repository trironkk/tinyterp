## What it is

nanochat is "the simplest experimental harness for training LLMs." It is the successor to
nanoGPT (which only covered pretraining) and is designed to run on a single GPU node with
minimal, hackable code that covers **all major LLM stages**: tokenization, pretraining,
finetuning (SFT), reinforcement learning, evaluation, inference (with KV cache), and a
ChatGPT-like web UI.

The headline claim: you can train your own GPT-2-capability LLM — which cost ~$43,000 to train
in 2019 — for ~$48 (about 2 hours on an 8×H100 node), or closer to ~$15 on spot instances, then
talk to it in a familiar chat UI.

## The single complexity dial: `--depth`

nanochat is configured around one integer, `--depth` (the number of transformer layers). This
single dial automatically determines all other hyperparameters — width, number of heads,
learning-rate adjustments, training horizons, weight decays — so the resulting model comes out
**compute-optimal**. Sweeping depth yields a "miniseries" of compute-optimal models. GPT-2
capability is approximately depth 24–26 with current code; depth-12 (~GPT-1 size) gives ~5-minute
experimentation runs.

## Time-to-GPT-2 leaderboard

Inspired by [modded-nanoGPT](https://github.com/KellerJordan/modded-nanogpt), nanochat maintains
a "GPT-2 speedrun" leaderboard: wall-clock time on an 8×H100 node to reach GPT-2-grade capability
as measured by the DCLM CORE score (target 0.256525). Selected entries:

| # | time (hrs) | CORE | Description | Date |
|---|------------|------|-------------|------|
| 0 | 168 | 0.2565 | Original OpenAI GPT-2 checkpoint | 2019 |
| 1 | 3.04 | 0.2585 | d24 baseline | Jan 29 2026 |
| 3 | 2.76 | 0.2602 | bump total batch size to 1M tokens | Feb 5 2026 |
| 4 | 2.02 | 0.2571 | change dataset to NVIDIA ClimbMix | Mar 4 2026 |
| 5 | 1.80 | 0.2690 | autoresearch round 1 | Mar 9 2026 |
| 6 | 1.65 | 0.2626 | autoresearch round 2 | Mar 14 2026 |

The primary metric is wall-clock "time to GPT-2." At ~$3/GPU/hr an 8×H100 node is ~$24/hr.

## Pipeline and usage

- Uses [uv](https://docs.astral.sh/uv/) for dependency management (`uv sync --extra gpu|cpu`).
- The entire reproduce-and-chat pipeline is the single file `runs/speedrun.sh` (designed for an
  8×H100 node, ~3 hours). Afterward, serve the web UI with `python -m scripts.chat_web`.
- Runs on a single GPU (omit `torchrun`; switches to gradient accumulation), or CPU/MPS via
  `runs/runcpu.sh` (shrinks the model drastically).

## Precision

No `torch.amp.autocast`. Precision is managed explicitly via a global `COMPUTE_DTYPE`
auto-detected by hardware (bf16 on SM 80+; fp32 on older CUDA, CPU, MPS; fp16 available with
GradScaler). Weights are stored in fp32 for optimizer precision and cast to `COMPUTE_DTYPE`
in the custom `Linear` forward pass.

## Code structure (selected)

The `nanochat/` package includes `gpt.py` (the GPT `nn.Module`), `tokenizer.py` (GPT-4-style BPE
wrapper), `engine.py` (inference with KV cache), `optim.py` (AdamW + Muon), `execution.py` (lets
the LLM run Python as a tool), and `core_eval.py` (DCLM CORE score). `scripts/` holds
`base_train`, `chat_sft`, `chat_rl`, `chat_web`, `tok_train`, etc. `tasks/` holds ARC, GSM8K,
HumanEval, MMLU, SmolTalk, and a spelling-bee task.

## Philosophy

"Accessibility is about overall cost but also about cognitive complexity." nanochat is "not an
exhaustively configurable LLM framework" — no giant config objects, model factories, or
if-then-else monsters — but "a single, cohesive, minimal, readable, hackable, maximally-forkable
strong baseline" that runs start-to-end and produces a ChatGPT model you can talk to.
